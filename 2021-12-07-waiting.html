Waiting for feedback

Sometimes an issue comes up where an enhancement has damaged the core codebase and therefore there are bugs on the main branch

Sometimes, basically removing the enhancement, or disabling it, will fix it

So, I oftentimes will make a PR to this effect and say, "here is a fix, it basically removes the enhancement, and fixes a bug. This is a bug users definitely will definitely run into, so we should probably merge this and release it so that they don't run into it". what I normally get in response to this is crickets

I can guess at some reasons why my coworkers are avoiding this issue maybe. For example

  • they don't want to "duplicate work" by undoing the enhancement on the master branch (and later redoing it)

  • they have pride in their feature and don't want to see it undone

  • they are trying to be sensitive to the feelings of the person who originally developed the enhancement, and avoid undoing it

  • they are not "satisfied" with the PR that removes the enhancement, and would rather see a fix to the enhancement

  • they are busy and don't have time to respond to requests to review, pings in chat messages, and tagged in pr comments

So, generally, there is very little feedback on these PRs, so I just have to sit around and feel anxious, and like a brat expecting people to care about this thing. Oftentimes it took me a fair amount of debugging to even make such a PR, since it is sometimes pretty unexpected why enhancement X would break some core thing Y so the silence makes me feel unappreciated in addition to all this.

Note that my team has generally not been great about code reviews so this is just an aspect to this, I fairly frequently merge my own PRs if they are small but in a case where it rolls back someone's work, I definitely feel like including them in the review process.

Overall, this is not really just about waiting for feedback. It's about remote work. It's about teamwork and communication and code review culture. It's also a pandemic, everyones busy, dropping balls, etc. If I find ways to improve, will update this post.
